# 나중에 수정할 목록 (테스트 이후)

테스트 기간 동안 기존 동작을 깨지 않으려고 미룬 변경들. 정식 배포 전에 처리.

## 신체 데이터 공유 동의를 한 곳으로 통일

**현재 동작 (그대로 둠):** 다른 멤버의 신체 수치(체중/골격근량/체지방률/측정 이력)가
보이려면 두 조건이 **모두** 참이어야 함.

1. `profiles.visibility = 'members'` — 프로필 시트의 "그룹 멤버에게 신체 정보 공개 허용"
   토글 (기본값 `private`)
2. `sharing_preferences.share_*` — 그룹 관리 > 내 공유 설정의 항목별 토글
   (`share_body` / `share_weight` / `share_body_fat`)

`get_bootstrap` RPC가 `visibility = 'members' AND share_*` 로 게이팅.

**문제:** 두 스위치가 서로 다른 화면에 있고 마스터(`visibility`)가 기본 꺼짐이라,
그룹 관리에서 항목별 공유만 켠 사용자는 신체 수치가 계속 차단됨. (음식·운동 로그는
`visibility`와 무관하게 `share_food`/`share_workout`만으로 공유돼서 동작이 불일치.)

**계획했던 수정 (테스트 후 적용):**
- `get_bootstrap`가 항목별 `share_*` 플래그만으로 판단하도록 변경
  (`visibility = 'members' AND` 조건 제거) — 음식·운동과 일관.
- 프로필 시트의 중복 "신체 정보 공개" 토글 제거, 그룹 관리의 항목별 설정을 단일 창구로.
- 로컬 검증까지 마친 상태였음 (`share_body`만 켜면 visibility가 private여도 노출).

**미룬 이유:** RPC(엔드포인트) 시그니처/동작을 바꾸면 이미 배포된 클라이언트·데이터와
맞물려 문제가 생길 수 있어, 테스트 기간에는 기존 계약을 유지.

**임시 우회:** 테스트 중에는 프로필 시트에서 "그룹 멤버에게 신체 정보 공개 허용"을 켜고,
그룹 관리에서 항목별 공유도 켜면 정상적으로 보임 (두 곳 모두 켜야 함).
